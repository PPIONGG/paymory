## Environment & gotchas (IMPORTANT ‚Äî Windows)
- **PostgreSQL 18**, native install, runs as Windows service `postgresql-x64-18` (auto-starts, leave it on). DB name: `paymory`. `psql` is **not on PATH** ‚Üí use `C:\Program Files\PostgreSQL\18\bin\psql.exe`.
- **DLL-lock gotcha:** the running `tsx watch` server locks Prisma's `query_engine-windows.dll.node`. Running `prisma generate`/`prisma migrate` while `npm run dev` is up fails with `EPERM ... rename`. **Fix: stop the dev server first** (kill ports 4000/5173), then run prisma commands, then restart. In Git Bash: `pid=$(netstat -ano | grep ":4000 " | grep LISTENING | awk '{print $5}' | head -1); taskkill //F //T //PID "$pid"` (note the `//` for flags).
- Bash tool cwd is the repo root. A full dev server may still be running in this session's background ‚Äî the next agent should kill ports and restart cleanly.
- `npm run dev` may report "2 critical vulnerabilities" ‚Äî unreviewed; check during Phase 7 (deploy), not blocking dev.
- psql + Windows console: avoid non-ASCII (e.g. em-dash) inside SQL string literals ‚Äî caused a `UTF8 0x97` error once. Set `PGCLIENTENCODING=UTF8` and use ASCII separators.

## Phase 4 ‚Äî what to build next (Dashboard + Monthly view)
Per BLUEPRINT ¬ß5. The big new concept: **Payment** records + **due** computation.
- **Payment endpoints** (new): mark an expense paid for a period (creates a `Payment` with snapshot `amountPaid`, `periodYear/Month`, `paidAt`, `markedByUserId`; unique per expense+period) and un-mark (delete). Scope to workspace.
- **Due logic** (compute, don't store): an expense is due in month M if MONTHLY, or if YEARLY and `dueMonth === M`. Monthly View lists due expenses for the selected month, grouped by due date, flags Upcoming/Overdue/Paid (Paid = a Payment exists for that period). Month total = actual sum due that month. Allow switching months.
- **Dashboard aggregates**: Monthly average = Œ£ monthly + Œ£(yearly/12); Yearly estimate = Œ£ monthly√ó12 + Œ£ yearly; total paid by each person (use payerType/payerUserId/splitPercent); shared vs personal; upcoming payments; active count; next due. Only ACTIVE expenses count toward totals (PAUSED/CANCELLED excluded).
- Wire into `HomePage` (currently a placeholder) and a new Monthly page; add nav links in `AppShell`.
- Remember the Decimal-as-string quirk: `amount` arrives as a string in JSON ‚Äî `Number(...)` it before math.
